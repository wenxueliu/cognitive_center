# Basic Memory 自动知识图谱交互命令模板

## 🚀 使用步骤

### 第1步：输入处理
```
🤖 基础命令格式：
"分析以下内容并自动转换为知识图谱：[你的内容]"

🔍 示例：
"分析以下内容并自动转换为知识图谱：我们决定采用微服务架构拆分单体应用，每个服务需要独立的数据库和用户认证系统，前端通过 API 网关统一访问后端服务"
```

### 第2步：自动检测和处理
系统将自动执行：
```
✅ 实体提取 (2-5个核心概念)
✅ 关系识别 (implements, requires, relates_to 等)
✅ 观察分类 ([decision], [requirement], [technique], [issue])
✅ WikiLink 创建 (memory:// 格式)
✅ 文件组织和存储
```

### 第3步：优化交互

#### 快速创建实体
```
💡 命令："创建实体：[实体名称] - 类型：[concept/system/person] - 描述：[简短描述]"
```

#### 快速建立关系  
```
🔗 命令："建立关系：[源实体] → [目标实体] - 类型：[relationship_type] - 说明：[关系描述]"
```

#### 快速记录观察
```
📝 命令："记录观察："[内容]" - 类型：[decision|requirement|technique|issue] - 上下文：[应用背景]"
```

## 📋 执行检查清单

### 自动验证项
- ☐ WikiLink 语法正确 (使用 memory:// 格式)
- ☐ YAML Frontmatter 完整 (包含 title, created, tags 等)
- ☐ 关系双向确认 (检查反向connectivity)
- ☐ 文件路径存在性 (确保目录结构正确)
- ☐ 标签一致性 (避免重复和冲突)

### 质量评估标准
- 📊 **实体覆盖率**: 提取的核心概念是否全面
- 🔗 **关系密度**: 实体间的连接是否充分
- 🏷️ **标签准确性**: 分类是否符合内容性质
- 🔄 **网络连通性**: 知识图谱是否形成有机网络

## 🔄 常见问题处理

### 实体重复处理
```
🤖 系统行为：
- 检测相同概念的多个表述
- 自动创建重定向关系
- 合并相似标签和属性
```

### 关系冲突解决  
```
⚖️ 冲突检测：
- 同一实体对的不同关系类型
- 互斥的技术方案
- 时间序列的决策变化
```

### 文件组织优化
```
🗂️ 智能分类：
- /output/zettelkasten/ - 原子化知识
- /output/create/ - 决策和创作内容
- /output/BASE/ - 系统化数据
- /input/ - 外部收集资料
```
